# 纳斯达克场外定投量化体系回测方案

## 一、 回测目标
验证量化信号对定投成本的优化能力，并评估在场外申购约束（T+N确认、申购限额、净值滞后）下的真实表现。

回测分为两步走：
1.  **信号有效性回测 (Version A)**：验证指标组合是否能精准识别补仓与风险区间。
2.  **真实执行回测 (Version B)**：加入每日 200 元上限、执行滞后及基金净值，看最终份额与收益。

---

## 二、 数据源配置

### 1. 核心指数 (建议从 FRED 获取)
| 指标名称 | FRED 代码 | 用途 |
| :--- | :--- | :--- |
| **纳指 100** | `NASDAQ100` | 计算趋势 (SMA200)、偏离度 |
| **VXN 波动率** | `VXNCLS` | 纳指期权恐慌程度 |
| **VIX 波动率** | `VIXCLS` | 全市场期权恐慌程度 |
| **等权纳指** | `NASDAQNDXE` | 计算 NDXE/NDX，识别结构背离 |
| **半导体指数** | `NASDAQSOX` | 计算 SOX/NDX，识别主线强弱 |

### 2. CNN 恐惧与贪婪指数
*   **获取途径**：FinHacker (2011 至今) 或 MacroMicro。
*   **特性**：0–100 综合情绪，建议包含 5 日均线以识别反转。

### 3. 基金净值数据
*   **必要性**：信号产生依赖指数，但**真实收益计算必须使用基金实际历史净值**。
*   **因素**：需考虑 QDII 基金的汇率影响、管理费及节假日确认错位。

---

## 三、 回测版本设计

### 版本 A：信号有效性验证 (无执行约束)
*   **标的**：QQQ (或 NDX 指数)。
*   **目的**：对比“策略定投”与“机械定投”在相同时间段内的成本差异。
*   **对照组**：
    *   固定频率定投 (Base)。
    *   无 CNN 版本 (Strategy v1)。
    *   有 CNN 完整版 (Strategy v2)。

### 版本 B：真实场景模拟 (核心版本)
*   **约束条件**：
    *   **每日限额**：单日订单金额 ≤ 200 元。
    *   **执行滞后**：模拟信号产生后 T+1, T+2, T+3 的申购确认。
    *   **状态机执行**：一旦触发补仓窗口（如 10 天），即便中途信号改变，也需完成或按周调整计划，而非每日剧烈跳变。
    *   **节假日处理**：非基金交易日不进行扣款，不追补双倍。

## 四、 关键字段与衍生计算
| 字段 | 计算逻辑 | 业务含义 |
| :--- | :--- | :--- |
| `dist_sma200` | `ndx / sma200 - 1` | 趋势偏离度 |
| `vxn_pctile` | `vxn.rolling(756).rank(pct=True)` | 滚动 3 年波动率分位 |
| `cnn_rebound` | `(cnn_ma5 > cnn_ma5.shift(1)) & (cnn < 25)` | 情绪修复信号 |
| `ndxe_repair` | `ndxe_ndx > ndxe_ndx.rolling(20).mean()` | 内部广度修复 |
| `buy_score` | 详见 Strategy V2 评分表 | 补仓强度评分 |
| `execution_date` | `signal_date + timedelta(days=N)` | 考虑滞后的实际成交日 |

---

## 五、 评价指标 (不仅仅看 ROI)
1.  **平均持有成本**：总投入 / 总份额 (最核心指标)。
2.  **累积份额数**：在同等总投入下，策略是否买到了更多份额。
3.  **最大回撤 (MDD)**：持仓市值从峰值的最大跌幅。
4.  **夏普/卡玛比率**：风险调整后的收益。
5.  **现金闲置率**：策略是否导致长期不买入从而踏空。
6.  **滞后敏感度**：在延迟 1 天与延迟 3 天的情况下，策略效果的衰减程度。

---

## 六、 样本分段建议
*   **2011 – 2019**：长牛市环境下的追高控制测试。
*   **2020**：疫情暴跌 V 型反转下的补仓反应测试。
*   **2022**：持续熊市下的弹药管理测试。
*   **2023 – 2026**：高波动、高集中度行情下的适应性测试。

---

## 七、 实施建议
1.  **参数敏感性分析**：不要寻找单一“最优”参数，而要确保在一组参数范围内（如 SMA 缓冲带 3%–7%）策略均有效。
2.  **未来函数检查**：严格确保 T 日的申购指令仅基于 T 日收盘前的已知信息。
3.  **先停后赎**：回测中应优先验证“暂停定投”的效果，再考虑“卖出战术仓”的增益。
